## Which Time is Better?

Since you are learning new, complex concepts (analysis and critical thinking):

- Morning (10:00 AM - 2:00 PM): Generally, the brain is most alert and ideal for tackling complex, analytical tasks like reading deep theory.
- Evening (8:00 PM - 11:00 PM): This time is often quieter and can boost creativity, making it good for coding and problem-solving where you apply the theory.

## üí° Brush-Up Tips for Interview Retention

The theoretical knowledge from the books must be translated into confident interview answers.

1. The "Teach It" Method

- Verbalize the Concepts: After reading a topic (like Closures), explain it out loud as if you were teaching a non-technical friend. This forces you to use precise, simple language, which is exactly what interviewers look for.
- Use Analogies: For complex topics, always have a simple analogy ready (e.g., the Event Loop is like a restaurant kitchen, the Call Stack is the line cook, the Callback Queue is the waiting area).

2. Practice Concept Implementation

Don't just code the examples; code the interview questions that rely on them.

- Closures: Implement a simple counter function that is only incrementable via an inner function.
- this: Implement your own version of the .bind() method.
- Async: Write a sequence of code with setTimeout, Promise.resolve(), and console.log() to predict the exact output order (testing the Microtask/Macrotask Queue).

3. Maintain a "Glossary of Confusing Terms"

Keep a simple note file with:

- Term: Hoisting
- Definition: The mechanism where variable/function declarations are moved to the top of their scope during compilation.
- Trap: Only the declaration is hoisted, not the initialization.

Review this short glossary every few days for quick recall.
